## 一、原始数据概览

## 二、预处理操作

### 2.1 维度缩减：20 → 3

根据项目需求文档——"支持按时间、菜品、服务维度分别统计情感分布"——只保留 **整体、菜品、服务** 三个维度。

被丢弃的 10 个标签列：

```
location_traffic_convenience, location_distance_from_business_district,
location_easy_to_find, price_level, price_cost_effective, price_discount,
environment_decoration, environment_noise, environment_space, environment_cleaness
```

### 2.2 标签聚合：多子标签 → 单粗标签

每个维度的多个子标签通过聚合函数合并为 1 个综合标签。

**聚合策略：pessimistic（保底策略 / 优先报忧）**

```
规则优先级：
  1. 若所有子标签均为 -2（未提及） → 聚合结果 = -2（未提及）
  2. 若任一子标签为 -1（负向）     → 聚合结果 = -1（负向）
  3. 若任一子标签为 1（正向）      → 聚合结果 = 1（正向）
  4. 否则（只剩中性）             → 聚合结果 = 0（中性）
```

**设计理由**：帮商家发现痛点——只要用户在菜品/服务的任一方面表达不满，该维度即判定为负向。例如：

> "服务员态度很好，但上菜太慢了"

原始标注：`service_waiters_attitude=1`, `service_serving_speed=-1`  
聚合结果：`service = -1`（负向），让商家关注到上菜速度问题

### 2.3 标签值重新映射

聚合后的标签从 `-2/-1/0/1` 映射到 `0/1/2/3`，适配 PyTorch `CrossEntropyLoss` 对类别标签的要求。

| 原始值 | 含义 | 映射后 | 说明 |
|--------|------|--------|------|
| -2 | 未提及 | **0** | 用户评价中未涉及该维度 |
| -1 | 负向 | **1** | 用户对该维度不满意 |
| 0 | 中性 | **2** | 用户提及但态度中性 |
| 1 | 正向 | **3** | 用户对该维度满意 |

**⚠️ 重要**：映射后的 `0` 对应"未提及"，不等于"中性"。这与常规三分类（负/中/正 → 0/1/2）不同，下游使用模型输出时需注意。

### 2.4 类别权重计算

由于聚合后仍存在正负样本不平衡（正向约为负向的 3~7 倍），自动计算类别权重用于训练时加权。

权重公式：$w_c = \frac{N}{K \times n_c}$

其中 $N$ = 总样本数，$K$ = 类别数(4)，$n_c$ = 类别 $c$ 的样本数。

**计算结果：**

| 维度 | 未提及(0) | 负向(1) | 中性(2) | 正向(3) |
|------|-----------|---------|---------|---------|
| overall | 17.52 | 2.55 | 1.31 | 0.36 |
| service | 0.83 | 1.65 | 2.22 | 0.58 |
| dish | 8.10 | 1.60 | 1.45 | 0.39 |

- 负向样本权重显著高于正向（如 overall 中负向权重 2.55 vs 正向 0.36），使模型更关注少数类
- `service` 分布相对均衡，权重差异最小
- `overall` 中未提及仅 1498 条，获得最高权重 17.52

## 四、聚合效果分析

### 4.1 关键指标改善

```
                   聚合前（子标签平均值）              聚合后
                   ──────────────────                ──────
service 未提及率:   ~76.7%（最低40%, 最高94%）  →   30.3%  ✓
dish 未提及率:      ~53.0%（最低5%, 最高81%）   →    3.1%  ✓
overall 未提及率:   ~32.3%（最低2%, 最高63%）   →    1.4%  ✓
```

聚合操作使"未提及"比例大幅下降，同时保留了"未提及"作为有意义的类别（而非强行归入中性）。

### 4.2 为什么聚合有效

以 **服务维度** 为例，用户可能只提到服务的某个方面：

| 评价内容 | wait_time | attitude | parking | speed | **聚合结果** |
|---------|-----------|----------|---------|-------|-------------|
| "服务员态度很好" | -2 | 1 | -2 | -2 | **1 (正向)** |
| "上菜太慢了" | -2 | -2 | -2 | -1 | **-1 (负向)** |
| "环境不错" | -2 | -2 | -2 | -2 | **-2 (未提及)** |

4 个子标签取并集后，只要用户提到了服务的任意一个方面，该维度就会被激活——这比单标签二分类（"是否提到服务"）更贴近真实语义。

## 六、注意事项

1. **测试集无标签**：`test_processed.csv` 中所有 `*_label` 均为 0，因为原始测试集未公开标签。模型评估请使用验证集。

2. **未提及 ≠ 中性**：标签 `0` 表示"用户未提及该维度"，标签 `2` 才是"中性"。下游统计时务必区分二者。

3. **可复现**：预处理脚本可重复执行，修改 `config.py` 中的 `AGGREGATION_STRATEGY` 可切换聚合策略（支持 `pessimistic` / `any_mentioned` / `strict` 三种）。

4. **去重**：预处理脚本内置了基于 `content` 列的去重逻辑（当前训练集无重复，未触发）。

5. **编码**：输出 CSV 使用 `utf-8-sig` 编码，Excel 可直接正确打开中文。
