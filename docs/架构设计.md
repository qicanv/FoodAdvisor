# FoodAdvisor 系统架构设计

## 1. 文档说明

本文档用于说明 FoodAdvisor 智能探店系统的总体技术架构、模块职责、服务调用关系和本地开发环境约定。

当前版本为 V0.1，主要用于团队前期并行开发。后续如发生接口、数据库或部署方式调整，应同步更新本文档。

---

## 2. 项目简介

FoodAdvisor 是一个基于大模型、检索增强生成和评论分析技术的智能探店系统。

系统面向普通用户提供自然语言探店、商家推荐、推荐解释、推荐反馈等功能；面向运营人员提供商家管理、评论分析、反馈统计和系统运行监控等功能。

## 4. 总体架构

系统采用前后端分离、业务后端与 AI 服务分离的架构。

```text
┌───────────────────────────────┐
│        Vue 3 Web 前端          │
│ 用户端 / 管理端 / 数据可视化   │
└───────────────┬───────────────┘
                │ HTTP / JSON
                ▼
┌───────────────────────────────┐
│       Spring Boot 业务后端      │
│ 用户、商家、会话、推荐、反馈    │
│ 权限、限流、统计、接口聚合      │
└───────┬───────────┬───────────┘
        │           │
        │           │ HTTP / JSON
        │           ▼
        │   ┌──────────────────────┐
        │   │    FastAPI AI 服务    │
        │   │ RAG、检索、评论分析   │
        │   │ 大模型调用、结果生成  │
        │   └──────────┬───────────┘
        │              │
        ▼              ▼
┌──────────────┐  ┌──────────────┐
│ PostgreSQL   │  │ OpenSearch   │
│ 正式业务数据 │  │ 文本与向量索引│
└──────────────┘  └──────────────┘
        │
        ▼
┌──────────────┐
│ Redis        │
│ 限流与缓存   │
└──────────────┘
```

## 5. 服务调用关系

统一调用方向如下：

```text
Vue → Spring Boot
Spring Boot → PostgreSQL
Spring Boot → Redis
Spring Boot → FastAPI
FastAPI → OpenSearch
FastAPI → 大模型 API
```

约束如下：

1. Vue 前端不直接访问 PostgreSQL、Redis 和 OpenSearch。
2. Vue 前端原则上不直接调用 FastAPI。
3. 所有面向用户和管理端的接口统一由 Spring Boot 暴露。
4. FastAPI 主要提供内部 AI 能力接口。
5. PostgreSQL 保存正式业务数据。
6. OpenSearch 保存用于搜索和 RAG 的索引数据。
7. Redis 不作为正式业务数据的唯一存储。

### 6.1 Vue 前端

负责：

- 商家推荐卡片

- 商家详情展示
- 管理后台

### 6.2 Spring Boot 后端

负责：

- 用户登录和身份校验
- 商家、菜品、评论等业务数据管理
- 会话和消息管理
- 推荐记录管理
- 推荐反馈管理
- 防重复提交
- Redis 接口限流
- 调用 FastAPI
- 统一返回格式
- 全局异常处理
- 管理后台统计接口

### 6.3 FastAPI AI 服务

负责：

- 用户问题理解
- 查询条件提取
- 知识库检索
- OpenSearch 全文和向量检索
- RAG 上下文构建
- 大模型调用
- 推荐解释生成
- 评论情感分析
- 差评原因提取
- 关键词和方面分析

### 6.4 PostgreSQL

负责保存：

- 用户
- 商家
- 菜品
- 评论
- 对话会话
- 对话消息
- 推荐记录
- 推荐商家明细
- 推荐反馈
- 管理统计所需正式数据

### 6.5 OpenSearch

负责保存：

- 商家检索文档
- 评论检索文档
- 菜品检索文档
- 知识块文本
- 文本向量
- 关键词和检索字段

### 6.6 Redis

负责：

- 用户接口访问频率限制
- IP 接口访问频率限制
- AI 接口调用限流
- 短时间重复请求拦截
- 热门数据缓存
- 临时会话状态
- 短期统计计数

## 8. Docker Compose 使用方式

首次拉取项目后：

```bash
copy .env.example .env
docker compose up -d
docker compose ps
```

正常情况下应看到：

```text
postgres      healthy
opensearch    healthy
redis         healthy
```

停止服务：

```bash
docker compose stop
```

重新启动：

```bash
docker compose start
```

停止并删除容器和网络：

```bash
docker compose down
```

注意：不要随意执行以下命令：

```bash
docker compose down -v
```

该命令会同时删除数据卷，可能导致本地数据丢失。

## 10. 数据流示例

用户发起探店请求时，数据流如下：

```text
1. 用户在 Vue 输入自然语言需求
2. Vue 调用 Spring Boot 对话接口
3. Spring Boot 验证用户并执行 Redis 限流
4. Spring Boot 保存用户消息到 PostgreSQL
5. Spring Boot 调用 FastAPI
6. FastAPI 理解需求并检索 OpenSearch
7. FastAPI 调用大模型生成推荐结果
8. FastAPI 将结构化结果返回 Spring Boot
9. Spring Boot 保存推荐记录和推荐明细
10. Spring Boot 将结果返回 Vue
11. Vue 展示推荐商家、理由和反馈按钮
```

---

## 11. 推荐反馈流程

```text
1. 用户对一次推荐选择满意或不满意
2. Vue 将 recommendationId 和反馈内容提交给 Spring Boot
3. Spring Boot 校验推荐记录和用户身份
4. PostgreSQL 唯一约束防止重复反馈
5. 用户修改反馈时更新原记录
6. 管理后台读取反馈统计结果
```

Redis可以用于短时间重复点击拦截，但正式反馈记录必须保存在 PostgreSQL 中。

---

## 12. 接口限流流程

```text
1. 请求到达 Spring Boot
2. 根据用户、IP和接口生成 Redis Key
3. Redis 增加计数并设置过期时间
4. 未超过限制则继续执行
5. 超过限制返回 HTTP 429
```

示例 Redis Key：

```text
rate_limit:user:12:ai_chat:202607140915
rate_limit:ip:127.0.0.1:login:202607140915
```

