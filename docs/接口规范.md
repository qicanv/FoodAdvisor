# FoodAdvisor 接口规范

## 1. 文档说明

本文档用于统一 FoodAdvisor 项目的接口路径、请求格式、响应格式、字段命名、权限控制、错误处理、分页方式、幂等规则，以及 Spring Boot 与 FastAPI 之间的内部调用协议。

当前版本为 **V0.2（MVP 可开发版）**，与《数据库设计 V0.2》保持一致。

本版本重点支持以下核心业务闭环：

```text
用户登录
→ 创建探店会话
→ 多轮需求提取与追问
→ 商家检索与推荐
→ 推荐理由与原始依据溯源
→ 用户反馈
→ 商家与评论数据导入
→ 评论情感和标签分析
→ 商家评价摘要
→ AI 调用日志和运行监控
```

接口发生变化时，应同步更新：

```text
docs/接口规范.md
docs/数据库设计.md
前端 API 封装
Spring Boot Controller / DTO
FastAPI Pydantic Schema
```

# 2. 服务与基础地址

## 2.1 本地开发地址

```text
前端：http://localhost:5173
业务后端：http://localhost:8080
AI 服务：http://localhost:8000
```

## 2.2 路径划分

面向前端和外部客户端的 Spring Boot 接口：

```text
/api
```

Spring Boot 调用 FastAPI 的内部接口：

```text
/internal
```

健康检查接口：

```text
/api/health
/health
```

## 2.3 调用原则

```text
前端 → Spring Boot → PostgreSQL / Redis
前端 → Spring Boot → FastAPI → OpenSearch / 大模型 API
```

前端不得直接调用 FastAPI 内部接口。

FastAPI 不直接修改用户、商家、反馈等正式业务数据。正式业务数据由 Spring Boot 负责落库。

```text
userId

reviewId

createdAt
updatedAt
```

禁止混用：


```text
/api/users

/api/chat/sessions
```

```text

```

接口中的枚举值统一使用大写下划线：

```text

SATISFIED
DISSATISFIED

NEGATIVE
```

# 5. 认证与权限

## 5.1 认证方式

对外接口使用 Bearer Token：

```http
Authorization: Bearer <token>
```

建议第一阶段采用 JWT。

## 5.2 用户角色

```text
USER       普通用户
MERCHANT   商家用户
OPERATOR   平台运营人员
ADMIN      平台管理员
```

## 5.3 权限原则

| 角色 | 主要权限 |
|---|---|
| USER | 探店对话、查看商家、提交推荐反馈 |
| MERCHANT | 查看并维护自己所属商家、菜品、评论分析 |
| OPERATOR | 评论审核、内容管理、运营统计 |
| ADMIN | 用户、商家、导入任务、系统监控和全部管理能力 |

商家用户访问商家资源时，Spring Boot 必须校验：

```text
当前用户是否存在有效的 merchant_members 关联
```

不能只根据请求中的 `merchantId` 判断权限。

# 6. 统一响应格式

## 6.1 成功响应

所有 Spring Boot 对外接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "requestId": "req-20260714-000001",
  "timestamp": "2026-07-14T09:30:00+08:00"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| code | integer | 业务状态码 |
| message | string | 提示信息 |
| data | object / array / null | 业务数据 |
| requestId | string | 请求追踪编号 |
| timestamp | string | 响应时间 |

## 6.2 失败响应

```json
{
  "code": 42201,
  "message": "参数校验失败",
  "data": {
    "errors": [
      {
        "field": "content",
        "message": "消息内容不能为空"
      }
    ]
  },
  "requestId": "req-20260714-000001",
  "timestamp": "2026-07-14T09:30:00+08:00"
}
```

## 6.3 FastAPI 内部响应

FastAPI 内部接口不强制使用 Spring Boot 的统一响应包装，但必须包含：

```text
requestId
status
```

示例：

```json
{
  "requestId": "req-20260714-000001",
  "status": "SUCCESS",
  "data": {}
}
```

# 8. 分页与排序规范

## 8.1 分页参数

```text
pageNum
pageSize
```

默认值：

```text
pageNum = 1
pageSize = 10
```

限制：

```text
1 <= pageSize <= 100
```

请求示例：

```http
GET /api/merchants?pageNum=1&pageSize=10
```

响应：

```json
{
  "code": 200,
  "message": "success",

    "total": 36,
    "totalPages": 4,
    "records": []
  },
  "requestId": "req-001",
  "timestamp": "2026-07-14T09:30:00+08:00"
}
```

## 8.2 排序参数

统一使用：

```text

sortDirection
```

示例：

```http
GET /api/merchants?sortBy=rating&sortDirection=DESC
```

允许值必须由接口白名单控制，禁止直接拼接用户输入生成 SQL。

# 9. 幂等与请求追踪

## 9.1 请求追踪编号

前端可以传递：

```http
X-Request-Id: req-20260714-000001
```

未传递时由 Spring Boot 自动生成。

Spring Boot 调用 FastAPI 时必须继续传递相同 `requestId`。

## 9.2 幂等键

以下接口必须支持幂等：

```text
POST /api/chat/sessions/{sessionId}/messages
PUT  /api/recommendations/{recommendationId}/feedback
POST /api/admin/import-tasks
POST /internal/rag/recommend
POST /internal/reviews/analyze
```

发送消息建议额外传递：

```http
Idempotency-Key: chat-1001-20260714-001
```

相同用户、相同接口、相同幂等键重复请求时：

- 已成功：返回第一次成功结果；
- 正在处理：返回 409 或当前处理状态；
- 上次失败：允许根据业务规则重试。

```json
    "recommendationId": 3002,
    "recommendations": [],
    "noMatchReasons": [
      "5公里内没有人均100元以内且评分不低于4.8的川菜商家"
    ],
    "adjustmentSuggestions": [
      {
        "field": "minimumRating",
        "currentValue": 4.8,
        "suggestedValue": 4.5,
        "message": "可将最低评分调整为4.5"
      },
      {
        "field": "maxDistanceKm",
        "currentValue": 5,
        "suggestedValue": 8,
        "message": "可将距离范围扩大到8公里"
```

要求：

1. 空白内容不得发送。
2. 同一幂等键不得新增第二条用户消息。
3. 每轮最多优先追问两个关键条件。
4. 新条件覆盖旧条件，未提及条件继续保留。
5. 新建会话不得继承上一会话条件。
6. `NO_MATCH` 与系统异常必须区分。
7. 返回商家必须存在于 PostgreSQL。
8. 停业、禁用和归档商家不得返回。

```http
PUT /api/recommendations/{recommendationId}/feedback

Content-Type: application/json
```


# 27. 内容清洗与切分接口

## 27.1 清洗与切分内容

```http
POST /internal/content/process
Content-Type: application/json
X-Internal-Token: <internal-token>
```

**功能说明**：对商家介绍、菜单描述和用户评价进行统一清洗和切分，去除无效字符、HTML标签、控制字符和无意义内容，统一日期和价格格式，并按配置规则切分为文本块。每个文本块保留完整的来源信息（merchantId、sourceType、sourceId、时间）。

请求：

```json
{
  "requestId": "req-001",
  "items": [
    {
      "merchantId": 1,
      "sourceType": "REVIEW",
      "sourceId": 101,
      "sourceTimestamp": "2026-07-14T09:30:00+08:00",
      "content": "<p>这家店<b>味道好</b>，2024年1月5日去的，人均50元。</p>",
      "metadata": {}
    },
    {
      "merchantId": 1,
      "sourceType": "MENU",
      "sourceId": 200,
      "sourceTimestamp": "2026-07-14T09:30:00+08:00",
      "content": "宫保鸡丁 - 38元\n麻辣豆腐 - 22元\n番茄蛋汤 - 15元",
      "metadata": {}
    }
  ],
  "cleanConfig": {
    "removeHtmlTags": true,
    "removeControlChars": true,
    "collapseSpaces": true,
    "removeMeaninglessPatterns": true,
    "normalizeDates": true,
    "normalizePrices": true,
    "customReplacements": {}
  },
  "chunkConfig": {
    "maxChunkLength": 512,
    "overlapLength": 64,
    "separator": "\n",
    "preserveSentenceBoundary": true
  }
}
```

请求参数说明：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| requestId | string | 否 | 请求追踪ID |
| items | array | 是 | 待处理内容列表，最多500条 |
| items[].merchantId | int | 是 | 商家编号 |
| items[].sourceType | string | 是 | 来源类型：MERCHANT_INTRO / MENU / REVIEW |
| items[].sourceId | int | 是 | 原始记录编号 |
| items[].sourceTimestamp | string | 否 | 原始内容时间（ISO格式） |
| items[].content | string | 是 | 待处理文本 |
| items[].metadata | object | 否 | 附加元数据 |
| cleanConfig.removeHtmlTags | bool | 否 | 去除HTML标签及实体，默认 true |
| cleanConfig.removeControlChars | bool | 否 | 去除控制字符，默认 true |
| cleanConfig.collapseSpaces | bool | 否 | 合并重复空格和换行，默认 true |
| cleanConfig.removeMeaninglessPatterns | bool | 否 | 去除无意义内容，默认 true |
| cleanConfig.normalizeDates | bool | 否 | 统一日期格式为YYYY-MM-DD，默认 true |
| cleanConfig.normalizePrices | bool | 否 | 统一价格格式为¥XX，默认 true |
| cleanConfig.customReplacements | object | 否 | 自定义替换规则 {旧文本: 新文本} |
| chunkConfig.maxChunkLength | int | 否 | 最大块长度（字符），默认512，范围[64,8192] |
| chunkConfig.overlapLength | int | 否 | 上下文重叠长度（字符），默认64，范围[0,512] |
| chunkConfig.separator | string | 否 | 切分优先分隔符，默认"\n" |
| chunkConfig.preserveSentenceBoundary | bool | 否 | 优先在句子边界切分，默认 true |

**清洗执行顺序**（不可变更）：
1. HTML标签移除 + 实体解码
2. 控制字符清除（保留换行符用于后续切分）
3. 日期标准化（2024年1月5日 / 2024/1/5 / 2024.1.5 → 2024-01-05）
4. 价格标准化（50元 / 50块钱 / RMB50 → ¥50）
5. 无意义内容移除（"该用户没有填写评价"、"默认好评"等）
6. 多余空格/换行合并
7. 自定义替换

**切分策略**：
- 优先按分隔符（默认 `\n`）拆分为段落
- 段落长度 ≤ maxChunkLength 时合并到当前块
- 段落超长时在句子边界（。！？!?）处切分
- 句子仍超长时强制按长度切分
- 块间保留 overlapLength 字符重叠

**容错机制**：单条数据处理失败时记录错误原因，不中断其他数据的处理。

**确定性**：相同输入和配置重复处理时，chunkId 保持一致（基于 SHA256 哈希生成）。

# 34. 评价举报接口（EPIC-08 故事5）

## 34.1 提交举报

```http
POST /api/reports
Authorization: Bearer <token>
Content-Type: application/json
```

请求：

```json
{
  "reportedReviewId": 101,
  "merchantId": 1,
  "reason": "ADVERTISING",
  "description": "该评价包含推广链接"
}
```

`reason` 可选值：

```text
ADVERTISING      广告引流
FALSE_REVIEW     虚假评价
MALICIOUS_ATTACK 恶意攻击
SEXUAL_OR_VULGAR 色情低俗
PRIVACY_LEAK     泄露隐私
OTHER            其他
```

`description` 选填，最多 500 字。

响应：

```json
{
  "code": "SUCCESS",
  "message": "举报已提交",
  "data": {
    "id": 1,
    "reporterUserId": 5,
    "reportedReviewId": 101,
    "merchantId": 1,
    "reason": "ADVERTISING",
    "description": "该评价包含推广链接",
    "status": "PENDING",
    "createdAt": "2026-07-21T09:30:00+08:00"
  }
}
```

错误场景：

```text
未选原因：400 INVALID_REASON
同一用户对同一评价已有未处理举报：409 ALREADY_REPORTED
未登录：401 UNAUTHORIZED
被举报评价不存在：404 REVIEW_NOT_FOUND
```

## 34.2 我的举报列表

```http
GET /api/reports/my?pageNum=1&pageSize=20&status=PENDING
Authorization: Bearer <token>
```

支持筛选：

```text
status: PENDING / RESOLVED / REJECTED（可选）
```

响应：

```json
{
  "code": "SUCCESS",
  "message": "Request succeeded",
  "data": {
    "pageNum": 1,
    "pageSize": 20,
    "total": 3,
    "records": [
      {
        "id": 1,
        "reportedReviewId": 101,
        "merchantId": 1,
        "merchantName": "川味小馆",
        "reviewSummary": "味道很好，但是周末排队时间太长...",
        "reason": "ADVERTISING",
        "reasonText": "广告引流",
        "description": "该评价包含推广链接",
        "status": "PENDING",
        "statusText": "待处理",
        "resolution": null,
        "createdAt": "2026-07-21T09:30:00+08:00",
        "handledAt": null
      }
    ]
  }
}
```
