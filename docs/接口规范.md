# FoodAdvisor 接口规范

## 1. 文档说明

本文档用于统一 FoodAdvisor 项目的接口路径、请求格式、响应格式、字段命名和错误处理方式。

当前版本为 V0.1，主要用于团队前期并行开发。接口实现发生变化时，应同步更新本文档。

---

## 2. 基本约定

### 2.1 协议

接口统一使用：

```text
HTTP / HTTPS
JSON
UTF-8
```

### 2.2 服务地址

本地开发环境：

```text
前端：http://localhost:5173
业务后端：http://localhost:8080
AI服务：http://localhost:8000
```

### 2.3 路径规范

面向前端的 Spring Boot 接口统一使用：

```text
/api
```

内部 AI 服务接口统一使用：

```text
/internal
```

健康检查接口可以使用：

```text
/health
```

## 3. 命名规范

### 3.1 JSON 字段

接口 JSON 字段统一使用小驼峰：

```text
userId
merchantId
reviewId
sessionId
messageId
recommendationId
feedbackType
createdAt
updatedAt
```

禁止在同一接口体系中混用：

```text
merchant_id
shopId
merchantID
```

### 3.2 URL

URL 使用小写字母和复数名词：

```text
/api/users
/api/merchants
/api/reviews
/api/recommendations
```

多单词使用短横线：

```text
/api/recommendation-feedback
```

### 3.3 时间格式

接口时间统一使用 ISO 8601：

```text
2026-07-14T09:30:00
```

如包含时区：

```text
2026-07-14T09:30:00+08:00
```

---

## 4. 统一响应格式

所有 Spring Boot 对外接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| code | integer | 业务状态码 |
| message | string | 提示信息 |
| data | object / array / null | 返回数据 |

成功示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1,
    "username": "demo"
  }
}
```

失败示例：

```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null
}
```

## 5. HTTP 状态码与业务码

| HTTP状态码 | 业务码 | 含义 |
|---:|---:|---|
| 200 | 200 | 请求成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未登录或登录状态失效 |
| 403 | 403 | 无权限访问 |
| 404 | 404 | 资源不存在 |
| 409 | 409 | 重复提交或数据冲突 |
| 422 | 422 | 参数校验失败 |
| 429 | 429 | 请求过于频繁 |
| 500 | 500 | 服务器内部错误 |
| 503 | 503 | 下游服务不可用 |

## 6. 分页规范

请求参数：

```text
pageNum
pageSize
```

示例：

```http
GET /api/merchants?pageNum=1&pageSize=10
```

响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "total": 36,
    "records": []
  }
}
```

默认值：

```text
pageNum = 1
pageSize = 10
```

建议最大：

```text
pageSize <= 100
```

## 11. 推荐反馈接口

### 11.1 提交或更新推荐反馈

```http
PUT /api/recommendations/{recommendationId}/feedback
Authorization: Bearer <token>
Content-Type: application/json
```

请求：

```json
{
  "feedbackType": "SATISFIED",
  "content": "推荐结果符合我的预算和口味"
}
```

`feedbackType` 可选值：

```text
SATISFIED
DISSATISFIED
```

响应：

```json
{
  "code": 200,
  "message": "反馈提交成功",
  "data": {
    "feedbackId": 4001,
    "recommendationId": 3001,
    "feedbackType": "SATISFIED",
    "content": "推荐结果符合我的预算和口味",
    "updatedAt": "2026-07-14T09:35:00"
  }
}
```

接口采用 `PUT`，表示同一用户对同一次推荐只有一条当前反馈。

再次提交时更新原反馈，不新增重复记录。

### 11.2 获取当前用户的推荐反馈

```http
GET /api/recommendations/{recommendationId}/feedback
Authorization: Bearer <token>
```

无反馈时：

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

## 14. 限流规范

需要限流的接口包括：

```text
POST /api/auth/login
POST /api/chat/messages
PUT /api/recommendations/{recommendationId}/feedback
```

超过限制时返回：

```http
HTTP 429
```

响应：

```json
{
  "code": 429,
  "message": "请求过于频繁，请稍后再试",
  "data": null
}
```

限流维度可包括：

- 用户 ID
- IP 地址
- 接口名称
- 时间窗口

请求参数错误时：

```json
{
  "code": 422,
  "message": "参数校验失败",
  "data": {
    "errors": [
      {
        "field": "content",
        "message": "消息内容不能为空"
      }
    ]
  }
}
```

---

## 16. 服务异常处理

FastAPI 不可用时，Spring Boot 返回：

```json
{
  "code": 503,
  "message": "AI服务暂时不可用，请稍后重试",
  "data": null
}
```

OpenSearch 不可用时，FastAPI 应返回明确错误，不允许伪造推荐结果。
